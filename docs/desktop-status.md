# fastgit Desktop 开发状态

> 更新时间：2026-07-02
> 分支：`codex/wails3`

## 1. 当前目标

为 `fastgit` 提供桌面客户端能力，并保持 CLI 主链路不受影响：

- CLI 继续走 `main.go` + `bootstrap.Main()`
- Desktop 走 `desktop/` 子项目（Wails v3 + React 前端）

## 2. 进度快照

### 2.1 已完成（可用）

- 桌面端基础壳与任务链路可运行（`task desktop:dev` / `task desktop:run`）。
- 前端升级到 Vite 8，Node 版本校验已接入 Task（`>=20.19` 或 `>=22.12`）。
- 建立“以 repo/project 为中心”的全局管理：
  - 项目命名空间（添加/切换/移除）
  - 项目级默认值（`defaultBaseBranch` / `defaultRemote`）
  - 会话级 GitHub Token 设置与状态检测
- 7 个资源模块已接入 SDK 调用（非命令字符串解析）：
  - `repo`、`remote`、`branch`、`worktree`、`issue`、`pr`、`tag`
- 管理页统一为“列表主视图”交互：
  - 搜索 / 过滤 / 排序
  - 行操作与详情侧栏
  - 批量操作（按模块能力启用）
- 高风险“强制对齐”能力已接入：
  - `repo_force_sync`
  - `branch_force_sync`
  - `tag_force_sync`
  - 统一 `RESET` 二次确认
- `remote` 管理能力已补齐：
  - 列表、添加、编辑、重命名、删除、抓取、抓取全部
- SSH 拉取/推送路径已增强（go-git）：
  - 读取 `~/.ssh/config`
  - 支持 identity files 与 ssh-agent
  - known_hosts 文件可用性探测与回退
- 页面切换交互已修复：
  - 点击左侧模块/顶部 Tab 会自动触发该模块默认列表动作。

### 2.2 进行中（优化项）

- 管理页样式继续简化（减少视觉噪声、增强按钮层次）。
- 响应式压缩场景（窄宽度）持续打磨，避免控件拥挤。
- 各模块“表格 + 弹窗 CRUD”的一致性交互仍在统一中。

### 2.3 待办（下一阶段）

- 增加桌面端 e2e 回归用例（至少覆盖模块切换、列表刷新、关键写操作确认）。
- 梳理前端组件抽象层（复用列表工具栏、详情面板、批量操作条）。
- 统一资源列表字段规范，减少各模块显示差异。
- 增加发布说明与版本变更记录（Desktop 维度）。

## 3. 可用能力矩阵

| 模块 | 列表 | 新增 | 编辑 | 删除/关闭 | 执行类 |
| --- | --- | --- | --- | --- | --- |
| repo | 状态 | - | 暂存/撤销暂存 | 丢弃文件 | 拉取/推送/强制对齐 |
| remote | 列表 | 添加 | 编辑/重命名 | 删除 | 抓取/抓取全部 |
| branch | 列表 | 创建 | 切换 | 删除 | 强制对齐 |
| worktree | 列表 | 创建 | - | 删除 | - |
| issue | 列表 | 创建 | 查看 | 关闭 | - |
| pr | 列表/状态 | 创建 | 同步/查看 | 关闭 | 合并 |
| tag | 列表 | 创建 | - | - | 推送/强制对齐 |

## 4. 运行与验证

### 4.1 开发运行

```bash
task desktop:dev
```

### 4.2 构建验证

```bash
task desktop:verify
```

### 4.3 最小回归清单

1. 切换任意模块（侧栏和顶部 Tab）后，主面板会刷新为对应列表页。
2. 在列表页执行搜索/过滤/排序，数据与计数同步变化。
3. 对 `force_sync` 类动作，未输入 `RESET` 不可提交。
4. 设置会话 Token 后，Issue/PR 模块可正常查询。
