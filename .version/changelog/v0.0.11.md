# v0.0.11

## 新增

- 新增 `fastgit pull --hard`，支持通过 `fetch + reset --hard` 强制同步远端分支。

## 修复

- 修复 `pull` 冲突处理时编辑器命令带参数（如 `code -w`）无法正确执行的问题。
- 修复 `push` 被本地未提交改动误拦截的问题，并改为显式返回执行错误。
- 修复 `tag --fast` 在无历史 tag 场景下的交互崩溃问题。
- 修复 `tag` 版本排序比较器不严格导致的潜在不稳定行为。

## 变更

- 优化 `pull` 分支跟踪逻辑：优先使用已配置 upstream，无 upstream 时自动回退到 `origin/<branch>`。
- 优化 `tag` 发布流程：发布前自动同步远端 tags，并在本地/远端冲突时提前失败。
- 调整 `tag` 策略为安全模式：不再自动 `git add -A`、`commit --amend`、`force-with-lease`。
- 在启用 `gen_version` 时，新增 `.version/VERSION` 与目标 tag 的主版本一致性校验。
